如果用户使用的部分功能，需要输入游戏模式，会优先使用群聊绑定的游戏模式。用户绑定的游戏模式不生效。

::::: important 群管专属使用方法

!groupmode / gm (`:` ::gamepad:: 游戏模式) ( ::phone:: 群组号)

:::: tabs

@tab:active ::gamepad:: 游戏模式

可选。`:` 可省略。

默认移除群聊绑定的游戏模式。此时机器人在需要指定游戏模式时，会使用玩家绑定的游戏模式、其次是玩家主页的默认游戏模式。

游戏模式对应表参见：[成绩/绑定游戏模式 (SM)](./mode.md)

@tab ::phone:: 群组号

可选。默认则视作操作当前群聊。

所有输入的号码都会被当做群组号。包括 `group=`、`qq=`。

::::

:::: important 超管专属使用方法

!viewgroupmode / vm ( ::phone:: 群组号) (`#` ::file:: 页码)

::: tabs

@tab:active ::phone:: 群组号

可选。默认查看所有绑定过游戏模式的群聊。

所有输入的号码都会被当做群组号。包括 `group=`、`qq=`。

@tab ::file:: 页码

可选。`#` 可省略。默认第 1 页。

:::

::::

:::::

::: tip 提示

群聊绑定的游戏模式，会优先于玩家绑定的游戏模式生效。所以这个功能，应该仅用于需要==控制玩家讨论游戏模式==的群聊中。

游戏模式的获取先后顺序：

- 玩家查询时，通过 `:` 冒号设定的游戏模式
- 群聊绑定的游戏模式
- 玩家绑定的游戏模式
- 玩家主页上的默认游戏模式

:::

::: warning 注意

在极少部分功能里，群聊游戏模式不会生效（否则不符合大部分用户的期望），比如谱面/查询谱面信息 (M) 功能。

:::
